SQL Mastery Mind Map
3-Tier Architecture
1. Presentation Layer (UI)
• User interface management • Captures and displays user interactions • Sends user requests to application layer • Provides visual representation of data
2. Application Layer (Business Logic)
• Processes business rules and algorithms • Validates and transforms data • Mediates between UI and database layers • Implements core application functionality
3. Database Layer (Data Storage)
• Manages data persistence • Handles data retrieval and storage • Ensures data integrity and security • Supports complex query processing
Database Fundamentals
Data: Structured Information
• Organized collection of related information • Structured in tables, records, and fields • Supports efficient data management • Enables complex data manipulation
DBMS: Database Management System
• Software for database administration • Provides data creation and manipulation tools • Manages data storage and retrieval • Ensures data consistency and security
RDBMS: Relational DBMS
• Uses relational model for data organization • Data stored in interconnected tables • Supports SQL for data manipulation • Implements referential integrity
Codd's Rules
1. Information Rule
• All data must be representable as table values • Ensures data is logically structured • Supports comprehensive data representation
2. Guaranteed Access Rule
• Each data element must be accessible • Supports atomic data access • Prevents data isolation
3. Systematic Null Handling
• Provides consistent null value management • Supports null as a distinct data state • Ensures predictable null behavior
Data Types
CHAR
  • Fixed-length character data
  • Example: CHAR(10)
VARCHAR
  • Variable-length character data
  • Example: VARCHAR(50)
NUMBER
  • Numeric data
  • Example: NUMBER(10, 2)
DATE
  • Stores date and time
  • Format: DD-MON-YYYY
Constraints
NOT NULL
  • Ensures a column cannot contain null values
UNIQUE
  • Ensures all values in a column are unique
CHECK
  • Validates values based on a condition
Primary Key vs Foreign Key
Primary Key
  • Uniquely identifies a record in a table
  • Cannot have null values
Foreign Key
  • References a primary key in another table
  • Enforces referential integrity
SQL Statements
DDL (Data Definition Language)
  • CREATE: Creates a new table, view, or other object
  • ALTER: Modifies an existing database object
  • DROP: Deletes an object
DML (Data Manipulation Language)
  • INSERT: Adds new rows to a table
  • UPDATE: Modifies existing data
  • DELETE: Removes rows from a table
TCL (Transaction Control Language)
  • COMMIT: Saves changes permanently
  • ROLLBACK: Reverts changes
  • SAVEPOINT: Creates intermediate savepoints
DCL (Data Control Language)
  • GRANT: Gives permissions to users
  • REVOKE: Removes permissions
DQL (Data Query Language)
  • SELECT: Retrieves data from the database
Operators
Arithmetic Operators
  • +, -, *, /, %
Concatenation Operator
  • Concatenates two strings
  • Example: 'Hello' || 'World'
Comparison Operators
  • =, !=, >, <, >=, <=
Logical Operators
  • AND, OR, NOT
Special Operators
  • BETWEEN, IN, LIKE, IS NULL
Set Operators
  • UNION, INTERSECT, MINUS
Functions
Single-Row Functions
  • LENGTH(), CONCAT(), UPPER(), LOWER()
  • INITCAP(), REVERSE(), SUBSTR()
  • REPLACE(), INSTR(), TRIM()
  • LPAD(), RPAD(), MOD(), POW()
  • ROUND(), TRUNC()
Multi-Row Functions
  • MAX(), MIN(), COUNT()
  • AVG(), SUM()
When to Use WHERE, HAVING, and GROUP BY
WHERE Clause
  • Filters rows before grouping
HAVING Clause
  • Filters groups after grouping
GROUP BY Clause
  • Groups rows by specified columns
Joins
INNER JOIN
  • Returns rows with matching values in both tables
LEFT JOIN
  • Returns all rows from the left table and matching rows from the right
RIGHT JOIN
  • Returns all rows from the right table and matching rows from the left
FULL OUTER JOIN
  • Returns all rows from both tables
Subqueries
Single-Row Subquery
  • Returns one row
Multi-Row Subquery
  • Returns multiple rows
Nested Subquery
  • Contains one subquery inside another
Date functions
SYSDATE
  • Returns the current system date and time
  • Example: SELECT SYSDATE FROM dual;
CURRENT_DATE
  • Returns the current date based on the session's time zone
  • Example: SELECT CURRENT_DATE FROM dual;
SYSTIMESTAMP
  • Returns the system's current timestamp including date, time, and time zone
  • Example: SELECT SYSTIMESTAMP FROM dual;
TO_DATE
  • Converts a string into a date format
  • Syntax: TO_DATE('string', 'format')
  • Example: TO_DATE('2024-11-25', 'YYYY-MM-DD')
LAST_DAY
  • Returns the last day of the month for a given date
  • Example: SELECT LAST_DAY(SYSDATE) FROM dual;
ADD_MONTHS
  • Adds a specified number of months to a date
  • Syntax: ADD_MONTHS(date, number_of_months)
  • Example: ADD_MONTHS(SYSDATE, 2)
MONTHS_BETWEEN
  • Calculates the number of months between two dates
  • Syntax: MONTHS_BETWEEN(date1, date2)
  • Example: MONTHS_BETWEEN('2024-11-25', '2024-10-25')
NEXT_DAY
  • Returns the date of the next specified weekday after a given date
  • Syntax: NEXT_DAY(date, 'weekday')
  • Example: NEXT_DAY(SYSDATE, 'MONDAY')
